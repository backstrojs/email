# Internationalization (i18n) Guide

Complete guide for implementing multi-language email support with Backstro Email.

Because Astro templates run server-side, any Node.js i18n library works directly in the frontmatter.

---

## Simple Locale Prop Pattern

For projects that don't need a full i18n library, pass a `locale` prop and define messages inline:

```astro
---
// emails/WelcomeEmail.astro
import { Html, Head, Body, Container, Preview, Heading, Text, Button } from '@backstro/email';

type Locale = 'en' | 'es' | 'fr' | 'de';

interface Props {
  name: string;
  verificationUrl: string;
  locale?: Locale;
}

const { name, verificationUrl, locale = 'en' } = Astro.props;

const messages: Record<Locale, { subject: string; greeting: string; body: string; cta: string }> = {
  en: {
    subject: 'Welcome – verify your email',
    greeting: 'Hi',
    body: "Thanks for signing up! We're excited to have you.",
    cta: 'Verify Email',
  },
  es: {
    subject: 'Bienvenido – verifica tu correo',
    greeting: 'Hola',
    body: '¡Gracias por registrarte! Estamos emocionados de tenerte.',
    cta: 'Verificar correo',
  },
  fr: {
    subject: 'Bienvenue – vérifiez votre adresse',
    greeting: 'Bonjour',
    body: "Merci de vous être inscrit ! Nous sommes ravis de vous accueillir.",
    cta: 'Vérifier l\'adresse',
  },
  de: {
    subject: 'Willkommen – E-Mail bestätigen',
    greeting: 'Hallo',
    body: 'Danke für Ihre Anmeldung! Wir freuen uns, Sie dabei zu haben.',
    cta: 'E-Mail bestätigen',
  },
};

const t = messages[locale];
---

<Html lang={locale}>
  <Head />
  <Body style={{ backgroundColor: '#f3f4f6', fontFamily: 'sans-serif' }}>
    <Preview>{t.subject}</Preview>
    <Container style={{ backgroundColor: '#ffffff', margin: '40px auto', padding: '24px', maxWidth: '560px', borderRadius: '8px' }}>
      <Heading as="h1" style={{ fontSize: '22px', fontWeight: 'bold', color: '#111827', margin: '0 0 16px' }}>
        {t.greeting} {name},
      </Heading>
      <Text style={{ fontSize: '14px', lineHeight: '24px', color: '#374151', margin: '0 0 20px' }}>
        {t.body}
      </Text>
      <Button
        href={verificationUrl}
        style={{
          backgroundColor: '#0070f3',
          color: '#ffffff',
          padding: '12px 24px',
          borderRadius: '6px',
          textDecoration: 'none',
          display: 'block',
          textAlign: 'center',
          fontWeight: 'bold',
          boxSizing: 'border-box',
        }}
      >
        {t.cta}
      </Button>
    </Container>
  </Body>
</Html>
```

Render with a locale:

```ts
import { render } from "@backstro/email/render";
import WelcomeEmail from "./emails/WelcomeEmail.astro";

const html = await render(WelcomeEmail, {
  name: "Alice",
  verificationUrl: "https://example.com/verify",
  locale: "es",
});
```

## JSON Message Files

For larger projects with many strings, load translations from JSON files:

```
project/
├── emails/
│   └── WelcomeEmail.astro
└── messages/
    ├── en.json
    ├── es.json
    └── fr.json
```

```json
// messages/en.json
{
  "welcome-email": {
    "subject": "Welcome – verify your email",
    "greeting": "Hi",
    "body": "Thanks for signing up!",
    "cta": "Verify Email",
    "footer": "If you have questions, reply to this email."
  }
}
```

```json
// messages/es.json
{
  "welcome-email": {
    "subject": "Bienvenido – verifica tu correo",
    "greeting": "Hola",
    "body": "¡Gracias por registrarte!",
    "cta": "Verificar correo",
    "footer": "Si tienes preguntas, responde a este correo."
  }
}
```

```astro
---
// emails/WelcomeEmail.astro
import type { Props as BaseProps } from '../types';

interface Props {
  name: string;
  verificationUrl: string;
  locale: string;
}

const { name, verificationUrl, locale } = Astro.props;

// Dynamic JSON import works in Node/Astro SSR
const messages = (await import(`../messages/${locale}.json`)).default;
const t = messages['welcome-email'];
---

<Html lang={locale}>
  <Head />
  <Body style={{ backgroundColor: '#f3f4f6', fontFamily: 'sans-serif' }}>
    <Preview>{t.subject}</Preview>
    <Container style={{ backgroundColor: '#ffffff', margin: '40px auto', padding: '24px', maxWidth: '560px' }}>
      <Text style={{ fontSize: '14px', color: '#374151', margin: '0 0 16px' }}>
        {t.greeting} {name},
      </Text>
      <Text style={{ fontSize: '14px', color: '#374151', margin: '0 0 20px' }}>
        {t.body}
      </Text>
      <Button href={verificationUrl} style={{ backgroundColor: '#0070f3', color: '#fff', padding: '12px 24px', borderRadius: '6px', textDecoration: 'none', display: 'block', textAlign: 'center', boxSizing: 'border-box' }}>
        {t.cta}
      </Button>
      <Text style={{ fontSize: '12px', color: '#9ca3af', margin: '24px 0 0' }}>{t.footer}</Text>
    </Container>
  </Body>
</Html>
```

## next-intl

Best for Next.js projects. Works in Astro frontmatter because `createTranslator` is a pure function:

```sh
npm install next-intl
```

```astro
---
import { createTranslator } from 'next-intl';
import { Html, Head, Body, Container, Text, Button } from '@backstro/email';

interface Props {
  name: string;
  verificationUrl: string;
  locale: string;
}

const { name, verificationUrl, locale } = Astro.props;

const t = createTranslator({
  messages: (await import(`../messages/${locale}.json`)).default,
  namespace: 'welcome-email',
  locale,
});
---

<Html lang={locale}>
  <Head />
  <Body style={{ backgroundColor: '#f3f4f6', fontFamily: 'sans-serif' }}>
    <Container style={{ backgroundColor: '#ffffff', margin: '40px auto', padding: '24px', maxWidth: '560px', borderRadius: '8px' }}>
      <Text style={{ fontSize: '14px', color: '#374151', margin: '0 0 16px' }}>
        {t('greeting')} {name},
      </Text>
      <Text style={{ fontSize: '14px', color: '#374151', margin: '0 0 20px' }}>
        {t('body')}
      </Text>
      <Button
        href={verificationUrl}
        style={{ backgroundColor: '#0070f3', color: '#fff', padding: '12px 24px', borderRadius: '6px', textDecoration: 'none', display: 'block', textAlign: 'center', boxSizing: 'border-box' }}
      >
        {t('cta')}
      </Button>
    </Container>
  </Body>
</Html>
```

Message files follow the same structure as in the JSON pattern above.

## RTL Support

For right-to-left languages (Arabic, Hebrew, Persian), set `dir="rtl"` on `<Html>` and mirror layout styles:

```astro
---
const isRtl = ['ar', 'he', 'fa', 'ur'].includes(locale);
---

<Html lang={locale} dir={isRtl ? 'rtl' : 'ltr'}>
  <Head />
  <Body style={{ fontFamily: 'sans-serif', direction: isRtl ? 'rtl' : 'ltr', textAlign: isRtl ? 'right' : 'left' }}>
    <Container style={{ backgroundColor: '#ffffff', margin: '40px auto', padding: '24px', maxWidth: '560px' }}>
      <Text style={{ fontSize: '14px', color: '#374151' }}>
        {t.body}
      </Text>
    </Container>
  </Body>
</Html>
```

## Sending with Multiple Locales

```ts
import { render } from "@backstro/email/render";
import WelcomeEmail from "./emails/WelcomeEmail.astro";

const users = [
  { name: "Alice", email: "alice@example.com", locale: "en" },
  { name: "María", email: "maria@example.com", locale: "es" },
  { name: "Jean", email: "jean@example.com", locale: "fr" },
];

for (const user of users) {
  const html = await render(WelcomeEmail, {
    name: user.name,
    verificationUrl: `https://example.com/verify?email=${user.email}`,
    locale: user.locale,
  });

  await resend.emails.send({
    from: "Acme <noreply@yourdomain.com>",
    to: user.email,
    subject: subjects[user.locale] ?? subjects.en,
    html,
  });
}
```
